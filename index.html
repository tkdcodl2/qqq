데이터 적재 가이드 (Supabase)
① 일봉 백필(1회, 필수) — 모바일 추천 방법: Supabase 대시보드 → SQL Editor → 패키지의 backfill.sql 전체를 붙여넣고 Run. DB가 직접 야후에서 전 기간(TQQQ 상장 2010-02~) 일봉을 받아 market_data에 저장합니다. 사전에 Database → Extensions에서 http 확장을 Enable 하세요. 끝나면 행 수·기간이 표시됩니다.
예비 수단: ⓐ GitHub 앱 → Actions → backfill-market-data → Run workflow (사전에 저장소 Secrets에 SUPABASE_URL/SERVICE_KEY 등록) ⓑ 엣지 펑션 v4 배포 후 …/fetch-market-data?range=max 호출 ⓒ PC에서 node로 backfill-market-data.mjs 실행.

② 전략 설정 동기화(기기 간 동기화 필수): Supabase SQL Editor에서 alter table settings add column if not exists strategy jsonb; 를 한 번 실행하세요. 이 컬럼이 있어야 비중·파라미터(변동성 상한 targetVol·측정일수 volDays 포함) 전체가 Supabase에 저장되어 휴대폰·PC가 같은 값으로 동기화됩니다. 실행 후 원하는 기기에서 설정을 한 번 저장하면 다른 기기에도 반영됩니다. 컬럼이 없으면 이 브라우저(localStorage)에만 저장되어 기기마다 값이 달라집니다(화면 하단에 경고 표시).

③ SGOV 매매 허용(v12.0, 1회): 방어자산에 현금(SGOV)이 추가되어 trades 테이블의 티커 체크 제약을 갱신해야 합니다. Supabase SQL Editor에서 실행: alter table trades drop constraint if exists trades_ticker_check; alter table trades add constraint trades_ticker_check check (ticker in ('QQQ','QLD','TQQQ','SCHD','SGOV','CASH')); — 실행 전까지 SGOV 매수/매도 저장이 "trades_ticker_check" 오류로 거부됩니다(기존 데이터는 영향 없음).

④ 데이터 한계: QLD 2006-06 / TQQQ 2010-02 / SCHD 2011-10 상장 이전은 실데이터가 없어 합성 근사로 채웁니다(QQQ 일복리 레버리지·비용 모델 + SCHD 분기앵커 보간). 1999~까지 백테스트하려면 backfill.sql(v2 — QQQ만 있는 날도 행 생성)을 다시 실행하세요. 합성 구간이 포함되면 결과에 경고가 표시됩니다. 앱은 market_data를 최신부터 최대 12,000행까지 로드합니다.

TQQQ 래더 전략 — 균형형 v12.0 (디폴트)

철학: 평소엔 기계적으로 TQQQ 100% 적립. 고점·저점은 못 맞추니 — 과열 신호가 켜진 주마다 30%p씩 방어자산으로 미리 덜어내고, 추세가 확실히 깨졌을 때만 전량 피신, 회복이 확인되면 매주 10%p씩(추력 복귀는 35%p) 분할 재진입합니다. 여기에 변동성 상한을 덧씌워 시장이 출렁이면 TQQQ를 자동으로 줄입니다. 판단·동작은 일주일에 한 번뿐입니다.

방어자산 = SCHD 40% + 현금(SGOV) 60% (§35, v12.0). TQQQ에서 덜어낸 비중(1−w)을 이 비율로 나눠 담습니다. SCHD는 QQQ와 양의 상관이라 폭락 때 −19~21% 같이 빠지는데, 현금을 섞으면 방어 자본이 보존돼 재진입 실탄이 커집니다(닷컴·GFC 포함 전체 구간, 복귀·변동성·수확 개선과 합쳐 세후 IRR +1.5%p·최종 +34% — §35: 40~60% 어디든 비슷한 고원, 특정 국면 과최적화 아님). 별도 단계가 추가된 것이 아니라, 기존 모든 단계에서 "SCHD로"라고 하던 곳이 "SCHD 40+SGOV 60으로"로 바뀐 것입니다. 여기에 연말 비과세 한도 수확과 적립 스티어링(둘 다 신호·위험 불변)을 더해 세후 성과를 끌어올립니다.

TQQQ 단독적립은 무매도라 세금이 거의 없지만 MDD −99.5%(전체) 후 회복 불가였습니다. 인앱 백테스트(샌드박스)로 각 구간·설정을 직접 재확인하세요.

⚠️ 본 자료는 과거 데이터 기반 시뮬레이션이며 투자 조언이 아닙니다. 과거 성과가 미래를 보장하지 않으며, 레버리지 ETF는 변동성 끌림(decay)과 -90% 이상의 실제 손실 위험이 있고, 본 전략의 MDD -58%는 실전에서 견디기 매우 어려운 수준입니다. 모든 투자 책임은 본인에게 있습니다.
