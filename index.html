데이터 적재 가이드 (Supabase)
① 일봉 백필(1회, 필수) — 모바일 추천 방법: Supabase 대시보드 → SQL Editor → 패키지의 backfill.sql 전체를 붙여넣고 Run. DB가 직접 야후에서 전 기간(TQQQ 상장 2010-02~) 일봉을 받아 market_data에 저장합니다. 사전에 Database → Extensions에서 http 확장을 Enable 하세요. 끝나면 행 수·기간이 표시됩니다.
예비 수단: ⓐ GitHub 앱 → Actions → backfill-market-data → Run workflow (사전에 저장소 Secrets에 SUPABASE_URL/SERVICE_KEY 등록) ⓑ 엣지 펑션 v4 배포 후 …/fetch-market-data?range=max 호출 ⓒ PC에서 node로 backfill-market-data.mjs 실행.

② 전략 설정 동기화(선택): Supabase SQL Editor에서 alter table settings add column if not exists strategy jsonb; 실행 → 설정(비중·파라미터)이 기기 간 동기화됩니다. 컬럼이 없으면 이 브라우저(localStorage)에만 저장됩니다.

③ 데이터 한계: QLD 2006-06 / TQQQ 2010-02 / SCHD 2011-10 상장 이전은 실데이터가 없어 합성 근사로 채웁니다(QQQ 일복리 레버리지·비용 모델 + SCHD 분기앵커 보간). 1999~까지 백테스트하려면 backfill.sql(v2 — QQQ만 있는 날도 행 생성)을 다시 실행하세요. 합성 구간이 포함되면 결과에 경고가 표시됩니다. 앱은 market_data를 최신부터 최대 12,000행까지 로드합니다.

적응형 레짐 배분 전략

v5.0 하이브리드. 원전략의 두 강점 — 추세 ON에서 TQQQ 80%를 길게 보유해 수익을 극대화하고, 추세 OFF(200일선 아래)에선 레버리지를 전량 차단 — 에 개선 엔진인 저점 사이클(폭락 −26% 확인 → 공격 재진입 → 트레일링 스탑 −8%/+10% → 회복·정상화 종료)을 결합했습니다. 저점·반등과 정상이 같은 보유(TQQQ80/QLD20)를 공유하고, 같은 레버리지는 잠식이 적은 QLD로 통합해 단계 전환 시 종목 교체를 최소화했습니다.

5가지 단계와 대응
매수 추천 종목·비중은 이렇게 계산됩니다

매일 종가 기준으로 QQQ에서 4개 지표를 계산합니다: 고점 대비 낙폭(DD)·200일선 이격률·20일 모멘텀·저점 사이클(엔게이지) 상태. 그리고 아래 순서로 먼저 맞는 조건이 그날의 단계가 됩니다.

실행 — 단계가 바뀐 날 즉시 리밸런싱. 시뮬레이션 결과 리밸런싱 주기는 "단계 변경 시 즉시"가 최선이었습니다(연 40.3% > 매주 40.0% > 매월 38.4% > 분기 37.5%). 매일 단계만 확인하고, 단계가 바뀐 날에만 보유 전체를 새 단계의 목표 비중으로 맞춥니다. 신규 적립금은 항상 현재 단계의 목표 비중대로(부족한 종목 우선) 매수하며, 대시보드의 "+N주 매수/매도"가 목표 비중과 현재 보유의 차이를 주수로 환산한 값입니다. 트레일링 스탑(저점·방어)도 단계 변경으로 취급되어 발동·해제일에 즉시 리밸런싱합니다.
